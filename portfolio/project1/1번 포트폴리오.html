1. Player System (Inverse Kinematics)

포탈 통과 시 및 3인칭 렌더링 상황에서 마우스 조준 방향에 맞춰 플레이어 캐릭터의 상체 회전과 조준 자세가 물리에 기반하여 자연스럽게 정렬되도록 역운동학(IK) 보정 시스템을 구축했습니다.

  • AimController: 마우스 스크린 포인터의 변화량을 수신하여 월드 축 상의 가상의 표적(aimTarget) 좌표를 갱신합니다.
  • AimIK: 유니티 뼈대 구조 내에서 스파인(Spine) 및 어깨, 목 관절 등 상체 골격 회전 벡터에 보간 가중치를 대입하여 정교하게 제어합니다.

2. 내적(Dot Product) 기반 텔레포트 판정

기존 충돌 트리거 판정은 순간이동 진입 시 충돌 타이밍과 속도 편차로 인해 화면이 툭툭 끊기는 현상을 보였습니다. 이를 개선하고자 플레이어 오프셋 벡터와 게이트 면의 법선 벡터(transform.forward) 간의 내적 계산 부호가 프레임 사이에 반전되는 지점을 공간 관통 찰나로 연산하도록 설계했습니다.

Vector3 offsetFromPortal = travelerT.position - transform.position;
int portalSide = System.Math.Sign(Vector3.Dot(offsetFromPortal, transform.forward));
int portalSideOld = System.Math.Sign(Vector3.Dot(traveler.previousOffsetFromPortal, transform.forward));

// **경계선 관통 시점 판정** 내적값 부호의 전후 프레임 비교를 통한 경계선 관통 시점 판정
if (portalSide != portalSideOld)
{
    traveler.Teleport(transform, linkedPortal.transform, m.GetColumn(3), m.rotation);
}
의존성 결합 해제 성과
기존 구조에서는 플레이어 컨트롤러 스크립트 내부에서 순간이동 코드가 종속되어 있었습니다. 이를 공통 추상 구조인 PortalTraveler 컴포넌트로 분리 정립함으로써 상자, 물리 큐브, 투사체 등 리지드바디를 탑재한 월드 상의 모든 객체가 공통 통과하도록 확장성을 보증했습니다.

3. Matrix4x4 거울 반사 시점 매핑

포탈 게이트 내부 너머로 비치는 환경은 월드에 존재하는 가상 카메라가 실시간 텍스처 투사 렌더링을 가동한 결과물입니다. 입출구 포탈이 서로 비스듬하게 어긋나 있거나 회전되어 배치되더라도 일관된 플레이어 시점을 유지하기 위해, 행렬 공간 변환 계산식을 도출했습니다.

// 플레이어 카메라 행렬 획득
Matrix4x4 localToWorldMatrix = playerCam.transform.localToWorldMatrix;

// 상대 포탈의 월드To로컬 변환 후, 이 포탈의 로컬To월드로 반사 연산 처리
localToWorldMatrix = transform.localToWorldMatrix * linkedPortal.transform.worldToLocalMatrix * localToWorldMatrix;

// 동기화할 가상 카메라 트랜스폼에 즉시 대입
portalCam.transform.SetPositionAndRotation(
    localToWorldMatrix.GetColumn(3), 
    localToWorldMatrix.rotation
);

4. 렌더링 최적화 및 Frustum Culling

포탈 렌더링은 실시간으로 추가적인 뷰 렌더링을 수행하기 때문에 극심한 CPU/GPU 드로우콜 증가를 동반합니다. 불필요한 그리기 작업을 막기 위해 시야각 선별 최적화 필터를 도입했습니다.

  • Frustum Culling: GeometryUtility API를 접목하여 포탈 면적 Bounding Box가 현재 플레이어 카메라 시야 추대 평면(Frustum Planes) 6개 내부에 위치하는지 교차 분석합니다. 시야각 바깥에 있는 포탈은 드로우콜 호출을 원천 스킵합니다.
  • Recursion Limits: 마주보는 포탈 배치에서 카메라 렌더링이 재귀 연쇄 호출되어 프레임이 깨지는 것을 방어하기 위해 최대 2~3회로 재귀 깊이를 한계화하는 차단 알고리즘을 빌드했습니다.

5. 그래픽 클론 및 단면 픽셀 셰이더

물체가 포탈 경계를 넘어가는 동안 절반은 이쪽 포탈에, 나머지 절반은 저쪽 포탈에 온전히 묘사되어야 공간 연속성을 완성할 수 있습니다. 이를 위해 GraphicsClone(실시간 외형 복제체) 및 Pixel Discard Shader(평면 클리핑 셰이더)를 설계했습니다.

// 픽셀 셰이더 영역: 포탈 평면 뒷면의 픽셀은 드로잉 단계에서 Discard 처리
half4 frag(Varyings i) : SV_Target
{
    float3 adjustedCentre = sliceCentre + sliceNormal * sliceOffsetDst;
    float3 offsetToSliceCentre = adjustedCentre - i.worldPos;

    // clip 내장함수: 인자값이 음수이면 픽셀 연산을 즉시 폐기하고 그리지 않음
    clip(dot(offsetToSliceCentre, sliceNormal));

    half4 c = tex2D(_MainText, i.uv) * _Color;
    return c;
}

6. 운동 보존 및 한계 검토

포탈을 빠져나가는 텔레포트 도중에도 속력과 낙하 가속도(방향/크기)가 온전히 유지되어야 모멘텀 운동 효과가 극대화됩니다. 트래블러 오브젝트가 탈출할 때 linkedPortal의 회전 벡터를 기준으로 속도 방향을 전환시켰습니다.

// PortalTraveler.cs 내부 속도 및 회전 동기화
public virtual void Teleport(Transform fromPortal, Transform toPortal, Vector3 pos, Quaternion rot)
{
    transform.position = pos;
    transform.rotation = rot;
    
    if (rb != null)
    {
        // **물리 속도 유지 전달** 로컬 방향 벡터 전환 곱연산을 통해 물리 속도 유지 전달
        rb.velocity = toPortal.TransformDirection(fromPortal.InverseTransformDirection(rb.velocity));
    }
}
터널링(Tunneling) 한계점 회고
고속 낙하로 인해 중력 속도 임계치를 넘는 프레임의 경우, 한 프레임당 통과 거리가 포탈 게이트의 트리거 충돌 두께를 지나쳐 관동을 스킵하고 바닥 벽면에 박히는 터널링이 일부 감지되었습니다. 추후 Continuous 탐색 모드 연동 및 레이캐스트 소인(Sweeping) 충돌 동적 판정 로직을 덧대어 개선할 예정입니다.
