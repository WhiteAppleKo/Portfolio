1. 장르 분석 및 중앙 집중식 UI 설계

서바이버 로그라이크 장르의 두 가지 가치인 '성장의 축적'과 '물량의 쾌감'을 극대화하기 위해 기획을 수립하고 이에 수반되는 시각 피드백 가독성 저하를 방어하기 위한 인터페이스 설계를 진행했습니다.

  • 조작 편의성 극대화: 플레이어는 적 무리를 회피/이동하는 단순한 조작에 집중하며, 모든 공격은 정해진 주기에 맞춰 자동으로 조준 사출 처리됩니다.
  • 중앙 집중 UX 패턴: 수백 마리의 적이 화면을 뒤덮는 런타임 환경에서 시선이 외곽 체력바로 분산되어 오동작이 일어나는 현상을 방어하기 위해 모든 지표(체력, 경험치)를 캐릭터 외곽 원형 링(Ring) 형태로 통합 집중 렌더링했습니다.
  • 컬러 스와치 분리: 플레이어의 범위 및 경험치 등의 아군 속성은 네온 블루 테마를 부여하고, 적의 접촉 범위 및 공격 투사체는 네온 레드 테마로 분리 렌더링하여 가독성을 개선했습니다.

2. Data - Logic - Visual (DLV) 아키텍처 규칙

유닛 및 무기 컴포넌트의 빈번한 추가 시 구조적 병목을 방지하기 위해 데이터 모델, 상황 판단 로직, 가시 렌더러를 3중 분리했습니다.

  • PureData & PureDataBase: 게임에 로딩되는 불변의 정적 데이터셋(SO, CSV 파싱)입니다. 생성 이후 런타임 데이터 오염을 예방하기 위해 모든 필드는 readonly로 구성되며 데이터의 무결성을 손상하지 않는 Logic-Free 규칙을 따릅니다.
  • RuntimeData: 게임 플레이 중 변동되는 데이터셋으로, PureData를 오직 읽기 전용으로 참조하며 수치 변경 시 C# 이벤트를 송신하는 옵저버 구조 및 자체 Tick() 수명주기 갱신 루프를 가집니다.
  • Logic: 의사결정 및 상황 판단 연산만 수행합니다. 물리 컴포넌트(Rigidbody, Collider)를 직접 참조하거나 수정하지 않고, 위치 조정 필요 시 ObjectVisual 인터페이스에 명령 하달만 담당합니다.
  • Visual & ObjectVisual (DIP): 상위 Logic에 구체적인 구현 방식(2D/3D, NavMesh/Translate)을 노출하지 않는 추상 데코레이터 인터페이스로 연계하여 결합도를 낮추었습니다.

3. 3축 Cube 육각형 좌표계 (HexGrid) 연산

사각형 그리드는 대각선 방향 거리가 직선 대비 멀어져 사거리 및 인접 연산 시 오차가 발생합니다. 주변 6개 타일 방향으로 물리 연산 거리가 완전히 동일하게 매핑되는 육각형 그리드를 적용하고, 삼각함수를 제거해 정수 연산만으로 거리를 구속하도록 3축 Cube 좌표계를 코드로 설계했습니다.

// 2D 컬럼/행 인덱스를 3축 Cube 좌표계(Vector3Int)로 즉시 계산
private Vector3Int OffsetToCube(int col, int row)
{
    var q = col - (row - (row & 1)) / 2;
    var r = row;
    return new Vector3Int(q, -q - r, r); // **핵심 R&D** q + r + s = 0 공식 적용
}

// 3D 월드 상의 좌표를 지그재그 홀수/짝수 보정 연산을 통해 가장 가까운 육각 큐브 셀로 변환
public Vector3Int WorldToCube(Vector3 worldPos)
{
    float width = Mathf.Sqrt(3f) * hexRadius;
    float height = 2f * hexRadius * 0.75f;

    int row = Mathf.RoundToInt(worldPos.z / height);
    float xOffset = (row % 2 != 0) ? width * 0.5f : 0f;
    int col = Mathf.RoundToInt((worldPos.x - xOffset) / width);

    return OffsetToCube(col, row);
}
맨해튼 거리 공식 (GetHexDistance)
무거운 제곱근 및 삼각함수 연산을 완전히 배제하고 정수 합산 및 비트 나눗셈 연산만으로 최단 격자 거리를 빠르게 가져옵니다.
Distance = (Mathf.Abs(a.x - b.x) + Mathf.Abs(a.y - b.y) + Mathf.Abs(a.z - b.z)) / 2; // **임계 거리 판정**

4. 2중 탐색 범위 최적화 (OverlapHexGrid)

수백 마리 몬스터의 큐브 격자 거리를 프레임마다 대조하는 부하를 방지하기 위해 2단계 범위 검색 최적화를 구성했습니다.

// 물리 구체 검출과 기하 격자 거리 판정을 연계한 2중 탐색
public List<Collider> ScanTargets(Vector3 centerPos, int range, LayerMask targetLayer)
{
    List<Collider> validTargets = new List<Collider>();
    
    // 1단계: Physics.OverlapSphere로 가상의 넓은 후보군을 1차 고속 검출
    float hexWidth = Mathf.Sqrt(3f) * hexRadius;
    float searchRadius = (hexWidth * range) + hexWidth;
    Collider[] hits = Physics.OverlapSphere(centerPos, searchRadius, targetLayer); // **1단계: 물리 탐색**
    
    Vector3Int centerCube = WorldToCube(centerPos);
    
    // 2단계: 수집된 범위 후보군들을 순회하며 실제 3축 큐브 거리가 유효 범위(range) 이내인지 정밀 대조
    foreach (var hit in hits)
    {
        Vector3Int hitCube = WorldToCube(hit.transform.position);
        if (GetHexDistance(centerCube, hitCube) <= range) // **2단계: 정밀 필터링**
        {
            validTargets.Add(hit);
        }
    }
    return validTargets;
}

5. 무기(Weapon) 프레임워크 & 스탯 연산

공통 스탯 갱신 및 쿨타임 라이프사이클을 추상화하고, 무기별로 공격 로직만을 오버라이드 구현하도록 캡슐화했습니다.

public abstract class Weapon : MonoBehaviour
{
    protected RuntimeDataWeapon model; // 런타임 스탯 모델
    protected IWeaponVisualizer visuals; // 시각 피드백 연출 인터페이스

    public abstract void WeaponSettingLogic();
    
    public virtual void AttackLogic()
    {
        visuals.PlayAttackAnimation(); // 공통 비주얼 연출 재생
        model.SetCooldown(model.FinalAttackDelay); // 공통 쿨타임 타이머 작동
    }
}

WeaponModifier 계층적 스탯 산출

무기의 원본 SO 데이터(PureData) 훼손을 막고, 런타임 개별 버프(Local) 및 전역 버프(Global)를 계층 합산하여 실시간 스탯을 산출하고 이벤트를 호출합니다.

6. 기획 - 개발 데이터 연동 및 최적화

기획자가 수정한 데이터 시트를 자동으로 유니티 Resources 에셋으로 로드하고 매핑하는 자동화 설계를 적용했습니다.

6.1. WeaponID 8자리 명명 규칙

구분 자릿수 설명 예시 파싱값 (`05010002` 기준)
1~2자리 소속 원본 데이터 시트 분류 코드 `05` (5번 무기 시트 테이블)
3~4자리 무기 공격 메커니즘 고유 타입 `01` (광역 AoEWeapon 공격형)
5~6자리 무기 기본 획득 및 포인트 해금 경로 `00` (상시 해금 기본 무기)
7~8자리 그룹 카테고리 내의 개별 고유 일련번호 `02` (2번째 뱀서 무기 개체)
  • CSV Importer & 프리팹 연결 자동화: Excel/Notion으로 기획한 CSV 데이터 시트를 에디터 단에서 ScriptableObject 에셋 파일로 원클릭 로드하며, 적재 시 WeaponID를 Resources 폴더 내 프리팹 ID와 대조하여 유니티 컴포넌트 프리팹 링크를 자동으로 설정/연계합니다.
  • Audio Pooling 최적화: 피격/사망 몬스터가 0체력 도달 즉시 SetActive(false) 로 풀에 반납될 때, 사운드가 도중에 뚝 끊겨 타격감이 상실되는 전통적인 오디오 클립 유실 버그를 방어했습니다. 효과음 재생 책임을 몬스터 객체에서 완전히 격리하여 독립된 AudioPool Manager에서 완결 재생하도록 구현했습니다.
