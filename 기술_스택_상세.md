# 기술 스택 상세 설명서

본 문서는 프로젝트에서 사용된 핵심 라이브러리 3종 및 렌더링 관련 라이브러리 3종에 대한 상세 설명서입니다. 
모든 라이브러리는 빌드 단계 없이 브라우저에서 직접 로드하는 **CDN(Content Delivery Network) 방식**을 사용합니다.

---

## Part 1. 핵심 라이브러리 (Core UI & Style)

### 1. React 18 (CDN)
- **개요**: 컴포넌트 기반 사용자 인터페이스(UI) 구축을 위한 JavaScript 라이브러리.
- **프로젝트 내 역할**: 발표용 슬라이드 덱(`slides/deck.html`) 및 문서 레이아웃 쇼케이스(`showcase/index.html`)의 컴포넌트 구조화 및 상태(State) 관리.
- **CDN 사용법**:
  ```html
  <!-- React 및 ReactDOM CDN 로드 -->
  <script src="https://unpkg.com/react@18/umd/react.production.min.js"></script>
  <script src="https://unpkg.com/react-dom@18/umd/react-dom.production.min.js"></script>

  <div id="root"></div>

  <script>
    const { useState } = React;
    function App() {
      const [count, setCount] = useState(0);
      return React.createElement('button', { onClick: () => setCount(count + 1) }, `클릭: ${count}`);
    }
    const root = ReactDOM.createRoot(document.getElementById('root'));
    root.render(React.createElement(App));
  </script>
  ```
- **장점**: 빌드(npm/Vite) 없이 로컬 HTML 파일에서 React 컴포넌트 기반 UI 개발 가능.
- **단점**: 규모가 커질 경우 성능 최적화(Tree Shaking 등) 불가능, 코드 파일이 거대해짐.

### 2. Babel Standalone (CDN)
- **개요**: JSX 문법과 최신 ES6+ JavaScript 코드를 구형 브라우저에서도 동작하도록 변환해주는 실시간 트랜스파일러.
- **프로젝트 내 역할**: 로컬 환경에서 Node.js 빌드 단계 없이 HTML 내부 `<script type="text/babel">` 태그 안의 JSX 코드를 실시간으로 컴파일하여 브라우저에서 실행 가능하도록 지원.
- **CDN 사용법**:
  ```html
  <script src="https://unpkg.com/@babel/standalone/babel.min.js"></script>
  <div id="root"></div>

  <script type="text/babel">
    // JSX 문법 직접 사용 가능
    const Element = () => <h1>Hello, World!</h1>;
    const root = ReactDOM.createRoot(document.getElementById('root'));
    root.render(<Element />);
  </script>
  ```
- **장점**: 로컬 개발 시 복잡한 빌드 환경설정 불필요.
- **단점**: 브라우저 런타임에서 JavaScript 코드를 실시간으로 컴파일하므로 초기 로딩 및 렌더링 성능 저하 유발 (대규모 서비스 부적합).

### 3. Tailwind CSS 3 (CDN)
- **개요**: 클래스명 조합으로 빠른 UI 스타일링을 지원하는 유틸리티 우선(Utility-First) CSS 프레임워크.
- **프로젝트 내 역할**: 쇼케이스 페이지 및 발표 덱의 세부 레이아웃, 반응형 디자인, 컴포넌트 스타일 작성.
- **CDN 사용법**:
  ```html
  <script src="https://cdn.tailwindcss.com"></script>
  <script>
    // 테마 확장 커스텀 설정
    tailwind.config = {
      theme: {
        extend: {
          colors: {
            brand: '#5B6AD0'
          }
        }
      }
    }
  </script>

  <div class="p-6 max-w-sm mx-auto bg-white rounded-xl shadow-lg flex items-center space-x-4">
    <div class="text-xl font-medium text-brand">Tailwind CSS</div>
  </div>
  ```
- **장점**: 별도 CSS 파일 작성 불필요, 정해진 유틸리티 클래스로 통일감 있는 디자인 구현 속도 극대화.
- **단점**: CDN 버전 사용 시 사용하지 않는 클래스까지 포함된 무거운 파일을 로드하므로 페이지 로딩 성능 저하 (Production 단계에서는 PurgeCSS 빌드 권장).

## Part 2. 텍스트 및 다이어그램 렌더링 라이브러리

### 4. Marked
- **개요**: 마크다운(Markdown) 형식의 문자열을 HTML 코드로 변환해주는 가벼운 컴파일러/파서.
- **프로젝트 내 역할**: 가이드 본문 온사이트 리더(`guide.html`)에서 `src/*.md` 파일을 Fetch API로 불러온 뒤 브라우저 화면에 마크다운 문서를 동적으로 렌더링.
- **CDN 사용법**:
  ```html
  <script src="https://cdn.jsdelivr.net/npm/marked/marked.min.js"></script>

  <div id="content"></div>

  <script>
    const markdownText = '# 마크다운 제목\n\n- 리스트 1\n- 리스트 2';
    document.getElementById('content').innerHTML = marked.parse(markdownText);
  </script>
  ```
- **장점**: 경량 라이브러리로 처리 속도가 매우 빠르고 GitHub Flavored Markdown(GFM) 사양을 잘 준수함.
- **단점**: 단순 파서이므로 XSS(크로스 사이트 스크립팅) 보안 방지를 위해서는 별도의 살균(Sanitize) 라이브러리 혼용 필요.

### 5. Mermaid 11
- **개요**: 텍스트(마크다운 유사 문법) 기반으로 다이어그램과 플로우차트를 동적으로 그려주는 시각화 라이브러리.
- **프로젝트 내 역할**: 크래프팅 시스템 설계 설명용 클래스 다이어그램, 시퀀스 다이어그램 등을 이미지 파일(.png) 대신 텍스트 코드로 관리하며 웹상에 SVG 포맷으로 실시간 라이브 렌더링.
- **CDN 사용법**:
  ```html
  <script src="https://cdn.jsdelivr.net/npm/mermaid@11/dist/mermaid.min.js"></script>
  
  <div class="mermaid">
    graph TD
      A[시작] --> B{판정}
      B -- 성공 --> C[성공 완료]
      B -- 실패 --> D[실패 복구]
  </div>

  <script>
    mermaid.initialize({ startOnLoad: true, theme: 'default' });
  </script>
  ```
- **장점**: 다이어그램 수정 시 이미지 편집기 없이 텍스트 수정만으로 즉각 업데이트 가능, 텍스트 기반이라 용량이 매우 작고 고해상도 SVG로 출력되어 화질 깨짐 없음.
- **단점**: 다이어그램 구조가 복잡해질수록 렌더링 연산 비용이 크고, 문법 오류 시 화면에 다이어그램이 렌더링되지 않음.

### 6. Highlight.js
- **개요**: 다양한 프로그래밍 언어의 구문 강조(Syntax Highlighting)를 제공하는 클라이언트 사이드 라이브러리.
- **프로젝트 내 역할**: 마크다운 내 포함된 C# 예제 코드 스니펫(`pre code` 태그)을 감지하고 키워드, 문자열, 주석 등에 컬러 테마를 입혀 가독성 향상.
- **CDN 사용법**:
  ```html
  <link rel="stylesheet" href="https://cdn.jsdelivr.net/npm/@highlightjs/cdn-assets@11.9.0/styles/github-dark.min.css" />
  <script src="https://cdn.jsdelivr.net/npm/@highlightjs/cdn-assets@11.9.0/highlight.min.js"></script>

  <pre><code class="language-csharp">
    public class Player {
        public string Name { get; set; }
    }
  </code></pre>

  <script>
    // 페이지 내 모든 코드블록 자동 하이라이트
    hljs.highlightAll();
  </script>
  ```
- **장점**: 다양한 테마 CSS 지원, 180개 이상의 프로그래밍 언어 자동 감지.
- **단점**: 자동 감지 기능이 오동작할 수 있으므로 명시적으로 `class="language-xxx"`를 지정해 주어야 안전함.
